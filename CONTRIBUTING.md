# Contributing to Financial Research Agent

Thank you for your interest in contributing to the Financial Research Agent!

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or fix
4. Install dependencies: `pip install -r requirements.txt`
5. Make your changes
6. Test your changes locally with `streamlit run app.py`
7. Commit and push to your fork
8. Open a Pull Request

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/financial-research-agent.git
cd financial-research-agent

# Create virtual environment (recommended)
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Run the app
streamlit run app.py
```

## Code Style

- Follow PEP 8 guidelines for Python code
- Use type hints where practical
- Add docstrings to functions and classes
- Keep functions focused and reasonably sized

## Areas for Contribution

- **Bug fixes**: Found a bug? Please open an issue first, then submit a fix
- **Documentation**: Improvements to README, docstrings, or adding examples
- **New features**: Please open an issue to discuss before implementing
- **Tests**: Adding test coverage is always welcome
- **UI/UX improvements**: Enhancements to the Streamlit interface

## Reporting Issues

When reporting issues, please include:
- Description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Python version and OS
- Relevant error messages or logs

## Pull Request Process

1. Update documentation if needed
2. Test your changes locally
3. Ensure your code follows the existing style
4. Write a clear PR description explaining your changes
5. Link any related issues

## Questions?

Feel free to open an issue for questions or discussion.
